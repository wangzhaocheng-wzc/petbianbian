## 问题分析
- 304 Not Modified 属于正常的模块缓存握手（Vite/ESM），不是功能错误；真正导致“发布帖子页面不可见”的常见原因：
  1) 未登录：创建/编辑帖子需登录（frontend/src/pages/Community.tsx:26–34, 118–129；PostList 在没有 `user` 时不显示“发布”按钮，且点击会提示登录）。
  2) 前端 API 基础地址默认指向 `http://localhost:5000/api`（frontend/src/utils/constants.ts:1–2），而后端当前运行在 `5050`，导致社区列表/详情等接口请求错误或失败，从而页面空白或仅显示加载。
- 后端社区接口：列表/详情允许未登录（backend/src/routes/community.ts:20–24），创建/修改/点赞/评论需要登录（backend/src/routes/community.ts:25–37）。

## 修复方案
1) 配置前端 API 地址为后端端口：
   - 在 `frontend/.env.local` 设置：`VITE_API_URL=http://localhost:5050/api`
   - 说明：前端默认读 `VITE_API_URL`，否则回退到 `http://localhost:5000/api`（frontend/src/utils/constants.ts:1–2）。
2) 登录以显示发布入口：
   - 打开 `http://localhost:5173/login` 登录后，`useAuth` 会填充 `user` 并展示创建按钮（frontend/src/hooks/useAuth.tsx:61–71, 85–96）。
   - 验证：本地存储存在 `access_token` 与 `refresh_token`，并能通过 `/auth/me` 获取用户信息。 
3) 验证社区页面加载：
   - 打开 `http://localhost:5173/community`，确认列表请求 `GET /api/community/posts` 返回 200 并显示帖子；未登录也应能看到列表。
   - 进入详情页，确认 `GET /api/community/posts/:id` 与 `GET /api/community/posts/:id/comments` 正常。
4) 针对大量 304 的优化说明：
   - 304 是浏览器与开发服务器的条件请求响应，属于正常现象；不影响功能。
   - 若页面仍有频繁资源回退（例如占位图反复加载），优先检查对应图片 URL 是否 404，并在前端的 `img onError` 中加入一次性回退（避免反复重试）。
5) 资源跨源加载的兼容建议：
   - 若社区图片使用后端静态 `/uploads/community/...`，需保证后端 `BASE_URL` 指向 `http://localhost:5050`（backend/src/config/constants.ts:18 被前述 `.env` 覆盖），以及静态路由 `/uploads` 已设置 CORS 头（backend/src/server.ts:86–95）。
